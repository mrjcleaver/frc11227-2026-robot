# Build and Deploy

## Requirements

| Tool | Version | Notes |
|---|---|---|
| WPILib | 2026 | Includes VS Code, Java 17, and Gradle toolchain |
| FRC Game Tools | 2026 | Driver Station, roboRIO imaging tool |
| CTRE Phoenix Tuner X | Latest | For swerve config and motor IDs |
| PathPlanner | Latest desktop app | For editing autonomous paths |

WPILib installer: https://docs.wpilib.org/en/stable/docs/zero-to-robot/step-2/wpilib-setup.html

The WPILib installer sets up everything needed to build — a bundled JDK, Gradle wrapper, and VS Code with the WPILib extension. You do not need a separate Java installation.

---

## Opening the Project

Open the `frc11227-2026-robot/` folder in VS Code. The WPILib extension will detect `build.gradle` automatically.

Alternatively, use any IDE that supports Gradle (IntelliJ IDEA works well). Do not use a plain text editor — the Gradle wrapper and vendor JSON files must be intact.

---

## Build

### From VS Code
Press `Ctrl+Shift+P` → **WPILib: Build Robot Code**

### From terminal
```bash
./gradlew build
```

On Windows without WPILib's bundled shell:
```
gradlew.bat build
```

A successful build prints `BUILD SUCCESSFUL`. Errors appear inline with file and line number.

### What build does
- Compiles all Java source under `src/main/java/`
- Packages a fat JAR (all dependencies bundled) at `build/libs/`
- Does **not** deploy to the robot

---

## Deploy

The robot must be powered on and connected — either over USB, Ethernet, or Wi-Fi (robot radio).

### From VS Code
Press `Ctrl+Shift+P` → **WPILib: Deploy Robot Code**

### From terminal
```bash
./gradlew deploy
```

### What deploy does
1. Builds the JAR
2. Connects to the roboRIO at `roborio-11227-frc.local` (team number from `.wpilib/wpilib_preferences.json`)
3. Copies the JAR to `/home/lvuser/` on the roboRIO
4. Copies the contents of `src/main/deploy/` to `/home/lvuser/deploy/` — this includes PathPlanner path and auto files
5. Restarts the robot program

PathPlanner files are deployed automatically with the code. If you edit a path in PathPlanner, you must redeploy for the change to take effect on the robot.

## Simulate

The project has desktop simulation support enabled.

### From VS Code
Press `Ctrl+Shift+P` → **WPILib: Simulate Robot Code**

### From terminal
```bash
./gradlew simulateJava
```

Simulation opens a GUI with a Driver Station simulator and a field view. Useful for testing command logic and PathPlanner paths without the physical robot. Hardware (motors, encoders) is not simulated — commands will run but motors will not respond.

## Vendor Libraries

Vendor dependencies are declared in `vendordeps/` as JSON files. These are checked in to source control and should not be edited manually. To update a vendor library:

**From VS Code:** `Ctrl+Shift+P` → **WPILib: Manage Vendor Libraries** → **Check for updates (online)**

Vendor libraries in use:

| Library | Purpose |
|---|---|
| Phoenix6 | CTRE TalonFX motor control |
| REVLib | REV SparkMax motor control |
| PathplannerLib | Autonomous path following |

## Common Problems

**`Team number not found`**
Open `.wpilib/wpilib_preferences.json` and confirm `"teamNumber": 11227` is present.

**`Could not connect to roboRIO`**
- Confirm the robot is on and the radio is broadcasting
- Try USB connection instead of Wi-Fi
- Run `ping roborio-11227-frc.local` to check connectivity

**`BUILD FAILED` with vendor errors**
Run `./gradlew` with `--refresh-dependencies` to re-download vendor JARs:
```bash
./gradlew build --refresh-dependencies
```

**PathPlanner paths not found at runtime**
Confirm `.path` and `.auto` files are in `src/main/deploy/pathplanner/paths/` and `autos/` respectively, then redeploy. File names are case-sensitive and must exactly match the strings in `Autos.java`.

**`TunerConstants` compile errors after re-running Tuner X**
`generated/TunerConstants.java` is regenerated by CTRE Tuner X. If it changes, review the diff before committing — motor IDs and physical constants may have changed.
