## Subsystems

### Drivetrain (`CommandSwerveDrivetrain`)
Full holonomic swerve drive generated by CTRE Tuner X. Supports field-centric control, odometry with Kalman filtering, and vision pose correction via Limelight. PathPlanner is configured here with translation PID (10, 0, 0) and rotation PID (7, 0, 0).

### Shooter (`ShooterSubsystem`)
Dual flywheel shooter with independent left/right flywheels (each with a lead, follow, and feeder motor — 6 TalonFX total). Flywheel speed is interpolated from distance using a lookup table (47–73 RPS over 1.57–5.0 m). Feeder runs at 30 RPS.

| CAN ID | Motor |
|---|---|
| 16, 17 | Left flywheel lead, follow |
| 18 | Left feeder |
| 19, 20 | Right flywheel lead, follow |
| 21 | Right feeder |

**Flywheel speed table:**

| Distance (m) | Flywheel speed (RPS) |
|---|---|
| 1.57 | 47 |
| 2.76 | 55 |
| 3.00 | 57 |
| 3.13 | 60 |
| 3.80 | 64.5 |
| 5.00 | 73 |

Values between entries are linearly interpolated.

### Intake (`IntakeSubsystem`)
Roller intake with a pivot for deploying and retracting. Uses a SparkMax for the rollers (RIO bus, CAN ID 2) and a TalonFX for the angle motor (CANivore, CAN ID 14). Hard stop detection is via stator current limit (95 A). A jiggle motion (1 Hz, amplitude 0.06) is used to seat game pieces.

| Constant | Value |
|---|---|
| Rotate speed | 0.3 |
| Roller intake speed | 0.7 |
| Jiggle frequency | 1 Hz |
| Jiggle amplitude | 0.06 |

## Driver Controls (Xbox Controller, Port 0)

| Input | Action |
|---|---|
| Left stick | Translation (field-centric) |
| Right stick X | Rotation |
| A (hold) | Brake (lock wheels) |
| Y (hold) | Shoot sequence (brake + shoot + jiggle intake) |
| Right bumper (hold) | Run intake |
| Back | Raise intake |
| Start | Lower intake |
| Left bumper | Reset field-centric heading |

Operator controller is on port 1.

## Vision

The Limelight is used for two purposes:
1. **Distance estimation** — calculates distance to the hub AprilTag (ID 10 on the 2026 Rebuilt Welded field) to set flywheel speed.
2. **Aim correction** — `limelight_aim_proportional()` returns a rotational correction based on horizontal offset (TX). Used both in teleop (commented-out right bumper binding) and in autonomous via `aimAtTarget()`.

Pose estimates from the Limelight are fused into the drivetrain's Kalman filter via `addVisionMeasurement()` in `Robot.java`.
